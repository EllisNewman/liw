# Life in Weeks 头脑风暴

本文件用于汇总“Life in Weeks”项目的设计、功能和技术讨论，后续可据此整理为规格与实现文档。

---

## 1. 产品愿景

用直观的“生命周历”将时间可视化，帮助用户理解人生进度、规划目标，并记录重要里程碑。

**核心隐喻**  
一格代表一周（週セル / Shuu Seru），纵向或横向排列为整个人生（人生週カレンダー / Jinsei Shuu Calendar）。

---

## 2. 设计与交互灵感

### 2.1 布局形式
- **经典矩阵**：52 列 × N 行（按年分组），适合从上到下阅读。
- **横向时间轴**：每行一个年龄段（例如 0-9、10-19…），强调人生阶段（人生段階 / Jinsei Dankai）。
- **按季度分块**：每 13 周为一个视觉组块，便于拆分目标。
- **折叠/展开**：可折叠过去、展开未来，强调“现在周”。

### 2.2 视觉编码
- **颜色分层**：已过周/当前周/未来周使用不同色阶。
- **阶段色**：学龄、工作、退休等阶段用主题色。
- **密度提示**：通过色彩密度表达事件密集度或目标完成度。
- **网格细节**：格线强度随缩放变化，避免视觉噪点。

### 2.3 交互体验
- **当前周高亮**：自动定位到“现在周”。
- **悬停/点击**：显示该周对应的日期范围与事件。
- **缩放与聚焦**：支持放大到单年/单月视图。
- **时间快进**：拖动滑条或输入年龄快速跳转。

## 3. 核心功能设想

### 3.1 基础信息
- 出生日期（起点日 / Kiten-bi）
- 期望寿命（期待寿命 / Kitai Jumyou）
- 时区与周起始日（周一/周日）

### 3.2 生活记录与目标
- **里程碑**：毕业、工作、结婚、迁居等。
- **目标规划**：设置目标周、标记完成度。
- **习惯标记**：重复事件映射到周格。
- **反思回顾**：按年生成回顾视图。

### 3.3 分享与导出
- **只读分享链接**（可匿名）
- **导出图片/PDF**
- **海报模式**（极简视觉展示）

### 3.4 隐私与本地化
- **本地存储优先**（浏览器本地）
- **可选账号体系**（云同步）
- **数据加密**（客户端加密选项）

## 5. 技术讨论（方向性）

### 5.1 前端与渲染
- **CSS Grid**：适合 52×N 结构，语义明确。
- **Canvas/WebGL**：适合高密度数据与动画。
- **响应式布局**：移动端改为滚动式时间条或分段卡片。

### 5.2 状态与数据
- 本地存储：LocalStorage / IndexedDB
- 账号体系（可选）：OAuth + 轻量后端
- 数据格式：JSON（可导入导出）

### 5.3 性能与体验
- 懒加载/虚拟滚动：只渲染可视区域周格
- 渐进式渲染：先显示框架，再填充数据

### 5.4 国际化与可访问性
- 多语言标签与周起始日规则
- 色盲友好配色
- 键盘可访问（可逐周导航）

---

## 6. 风险与关键问题

- 视觉信息过载：如何在简洁与可读之间平衡？
- 用户隐私：默认本地存储是否足够？
- 目标与记录功能的复杂度：是否分层开放？

---
